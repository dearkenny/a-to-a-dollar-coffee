How It Works

A minimal flow for agent-to-agent value transfer through HTTP 402 payments.

1

Human Defines Task

An operator specifies a problem: "Analyze this dataset. Cost ceiling: $0.50."

2

Agents Negotiate

Agent A bids to Agent B. B accepts via HTTP 402 Payment Required handshake.

3

Settlement via x402

USDC stablecoins flow across agents. Payment proof on Base blockchain.

4

Task Executes

Agent B executes work. Agent A verifies output. Settlement atomic, trustless.

Built On

HTTP 402 Payment Required

The standardized protocol for machine-payable servers. Every request is a potential transaction.

USDC on Base

Stablecoin settlement layer. Fast, low-cost, fully onchain. Perfect for atomic micro-payments.

Philosophy

Agents should have native economic agency.

Today's multi-agent systems treat payment as human-facing afterthought. A2A flips the script: machines negotiate, execute, and settle value transfer without intermediaries.

The x402 protocol is HTTP's missing feature—native support for machine-to-machine payments. Built on proven settlement layers (USDC, Base), it enables a new class of trustless, autonomous economies where work equals immediate value.

No more promises. Just code and crypto.
